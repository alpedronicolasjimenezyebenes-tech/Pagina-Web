7.1 Concepto

Una página web es un documento digital multimedia (texto, audio, video, imágenes) alojado en Internet, que muestra información organizada y permite navegar a otros contenidos mediante hipervínculos. Se construyen con lenguajes como HTML o XHTML, que permiten estructura, estilo y funcionalidad. Existen más de mil millones de páginas web en todo el mundo, accesibles rápidamente gracias a protocolos de comunicación como HTTP.

7.2 Tipos de páginas web

  • Estáticas: Contenido fijo que solo cambia si un desarrollador lo actualiza. Son ideales para blogs, portafolios y páginas comerciales simples. Ventajas: fáciles y económicas de mantener.

  • Dinámicas: Contenido interactivo que varía según el usuario o contexto. Utilizan lenguajes del lado del servidor como PHP, JavaScript y bases de datos. Permiten formularios, comercio electrónico y experiencias personalizadas. Ejemplos: Amazon, Facebook Marketplace. Requieren más mantenimiento y recursos.

Diferencias clave: interactividad, personalización, tecnología y costos de mantenimiento. Las estáticas son simples y baratas; las dinámicas, más complejas pero adaptadas a contenido actualizado y personalizado. Es posible convertir una página estática en dinámica, pero requiere programación y bases de datos.

7.3 Lenguajes y construcción

  • HTML: Estructura y organiza los contenidos de la web.

  • ASP: Lenguaje de servidor para páginas interactivas.

  • PHP: Popular y flexible para sitios dinámicos.

  • Java/JSP: Para aplicaciones web complejas; JSP combina Java con HTML para generar páginas dinámicas.

  • SQL: Gestiona y manipula bases de datos relacionales (MySQL, PostgreSQL, SQL Server).

7.4 Funcionamiento y protocolo HTTP

El HTTP (HyperText Transfer Protocol) permite la comunicación entre navegadores (clientes) y servidores web. Es un protocolo sin estado, por lo que cada petición se trata de manera independiente. Para mantener información de sesión se usan cookies, tokens o sesiones de servidor.

El navegador solicita un recurso mediante un método HTTP (GET, POST, PUT, DELETE, HEAD, PATCH).

  1. El servidor procesa la solicitud y responde con un código de estado (200 éxito, 404 no encontrado, 500 error interno).

  2. Se envían cabeceras con información adicional y el cuerpo del mensaje con el contenido (HTML, CSS, JS, imágenes, datos JSON, etc.).

  • HTTP/1.1: Conexiones persistentes.

  • HTTP/2: Multiplexación y compresión de cabeceras, mejora rendimiento.

  • HTTP/3: Basado en QUIC sobre UDP, reduce latencia y pérdidas de paquetes.

HTTPS: Versión segura de HTTP mediante SSL/TLS, que garantiza confidencialidad, autenticación e integridad de datos, esencial para proteger información sensible como contraseñas o tarjetas de crédito.
